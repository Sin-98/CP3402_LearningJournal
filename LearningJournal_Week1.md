## Content Insights
This week helped me better understand how GitHub is used for version control and project management. Before this, I mainly saw GitHub as a place to upload files, but I learned that repositories are designed to track changes over time and support collaboration between developers.
I learned that Markdown is a lightweight formatting language commonly used for documentation. Using headings, bullet points, code blocks, and links makes files much easier to read and maintain.
I also learned how GitHub Pages can be used to host static websites directly from a GitHub repository. Through the publishing process, I learned that GitHub Pages automatically deploys the HTML and CSS files stored in the repository and generates a public website URL.

## Career/Employability/Learning Insights
This week made me realise how important GitHub is in the software and web development industry. Many internship listings and junior developer roles mention GitHub experience as a required or preferred skill because employers expect developers to understand version control and collaborative workflows.
While learning GitHub, I noticed that I understood concepts better when I practised them immediately after watching tutorials. Watching long videos without applying the concepts made it easy to forget steps. However, when I followed along myself, I learned much faster and became more confident.
I also realised that troubleshooting is an important part of learning technical skills. Sometimes I encountered issues when cloning repositories or pushing files, but solving those problems helped me understand the workflow more deeply than simply following instructions perfectly.
