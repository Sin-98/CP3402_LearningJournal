## Content Insights
This week I learned how software development teams organise projects before writing significant amounts of code. Instead of immediately starting implementation, the team first established communication channels, version control, project management tools, and a shared agreement on expectations. I realised that these organisational steps are essential because they reduce confusion and help everyone understand their responsibilities.
I also learned more about GitHub as a collaboration platform rather than just a place to store code. Features such as repositories, branches, commits, pull requests, and issue tracking help multiple developers work on the same project without overwriting each other's work. GitHub Projects showed me how development tasks can be organised into manageable pieces. Breaking a large project into smaller tasks makes progress easier to monitor and helps ensure that work is distributed fairly among team members.
Completing the group agreement highlighted the importance of defining expectations early. Discussing communication methods, availability, individual strengths, and responsibilities helps reduce misunderstandings later in the project.

## Career/Employability/Learning Insights
This week's practical felt much closer to how professional software development teams operate than previous weeks. Most employers expect developers to work collaboratively using tools such as GitHub, Slack, and Agile project boards rather than working independently. Learning how these tools fit together has given me a better understanding of industry workflows.
One insight I gained was that successful teamwork depends on transparency rather than simply working hard. By regularly committing code, updating tasks, and communicating progress, every team member can see how the project is developing. This reduces uncertainty and allows problems to be identified much earlier.
Researching software engineering job advertisements also reinforced how frequently employers mention collaboration skills alongside technical skills. Experience with Git, GitHub, Agile methodologies, issue tracking, and team communication appears in many junior developer roles. Although I already had some experience using GitHub individually, using it as part of a coordinated team required a different mindset because every change has the potential to affect other developers.
